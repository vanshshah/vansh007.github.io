My Projects

Twitter Replica | (Version 1,2,3)

Basic Functionalities

Build a "Prototype of Twitter application" using Node.js based server and it should provide the following functionalities.

a) Basic Users functionalities:
1. Sign up new user (First name, last name, Email and password)
2. Sign in existing user
3. Sign out. Sign Up should have first name, last name, Email and password.
In order to use the system, a user must sign in first to the system.

b) Users account should provide basic details such as:
1. About: User details, Work and education, contact info.
2. User can follow other users by searching their name or from suggestions provided.
3. Follower and following list. User should be able to see list of both..
4. User can choose a unique username to be displayed on profile.

c) Provide Tweet functionality.
1. User can tweet and retweet.
2. User should search for tweets using hashtag and user should search for any other user by their name.
e) Should perform connection pooling for database access implemented on custom code based on our own algorithm.
The Service also take care of exception handling mechanism to ensure the security of the application.

Version 1 :
• Developed Rest services using Node.js and DB as MYSQL .
• For UI part used Angular and Bootstrap .
• Wrote an algorithm for connection Pooling and compared load testing result with and without it for achieving perfomance .
• Testing for this project was done by JMeter, where 100,200,300,400,500 and 1000 concurrent users load was tested.

Version 2 :
• Developed Rest services using Node.js and DB as MongoDB .
• For UI part used Angular and Bootstrap .
• Reused an algorithm for connection Pooling and compared load testing result with and without it for achieving performance.
• Implemented Message Queuing through RabbitMQ. Followed Remote procedure calls in order to run functions on a remote computer and wait for the result.
• Testing for this project was done by JMeter, where 100,200,300,400,500 and 1000 concurrent users load was tested. Also tested using Mocha test Framework


Version 3 :
• Developed SOAP based web services using JAX-WS (Java API for XML Web Services) and Apache Axis.
• For UI part used Angular and Bootstrap .
